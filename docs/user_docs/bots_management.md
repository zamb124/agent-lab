# Управление ботами

Раздел "Боты" - это центр управления вашими ИИ-агентами. Здесь вы создаете, настраиваете и подключаете ботов к различным платформам.

## Что такое бот в Agent Lab

Бот в Agent Lab - это точка входа (Flow), которая объединяет:

- Агента или цепочку агентов
- Подключения к платформам (Telegram, WhatsApp, Web, AmoCRM, API)
- Конфигурацию и переменные
- Настройки LLM и базы знаний

## Создание бота

### Шаг 1: Создание

1. Перейдите в раздел "Боты"
2. Нажмите кнопку "+ Создать бота"
3. Заполните основные поля:
   - **Название** - понятное имя бота (например, "Бот поддержки")
   - **Описание** - для чего предназначен бот
   - **Entry Point Agent** - главный агент, который будет обрабатывать запросы

### Шаг 2: Настройка агента

В секции настройки агента укажите:

**Промпт агента**
Системный промпт определяет поведение бота. Пример:

```
Ты - бот технической поддержки компании.
Твоя задача:
- Отвечать на вопросы клиентов
- Быть вежливым и helpful
- Если не знаешь ответа - передавай человеку
```

**Локальные переменные агента**
Переменные, доступные только этому агенту. Формат JSON:

```json
{
  "company_name": "Моя Компания",
  "support_email": "support@example.com"
}
```

### Шаг 3: Настройка LLM

Выберите модель для вашего бота:

**Провайдер:**
- `openai` - ChatGPT (рекомендуется для начала)
- `yandex` - YandexGPT
- `gemini` - Google Gemini

**Модель:**
- OpenAI: `gpt-4`, `gpt-4-turbo`, `gpt-3.5-turbo`
- Yandex: `yandexgpt/latest`, `yandexgpt-lite/latest`
- Gemini: `gemini-1.5-pro`, `gemini-1.5-flash`

**Температура** (0-1):
- 0 - детерминированные ответы
- 0.7 - сбалансированная креативность
- 1 - максимальная креативность

### Шаг 4: Настройка RAG (База знаний)

RAG (Retrieval-Augmented Generation) позволяет боту искать информацию в загруженных документах.

**Параметры:**
- **Enabled** - включить RAG
- **Namespace Scope** - область поиска:
  - `flow` - только документы этого бота
  - `company` - все документы компании
- **Search Scopes** - где искать:
  - `flow` - в документах бота
  - `company` - в общих документах компании
- **Auto Index Messages** - автоматически индексировать сообщения

### Шаг 5: Переменные Flow

Глобальные переменные, доступные всем агентам в боте. Пример:

```json
{
  "api_url": "https://api.example.com",
  "timeout": 30,
  "max_retries": 3
}
```

### Шаг 6: Дополнительные настройки

**Timeout** - максимальное время выполнения (в секундах)  
**Max Retries** - количество попыток при ошибке

## Подключение к платформам

### Telegram

1. Создайте бота через [@BotFather](https://t.me/botfather)
2. Получите токен (вида: `123456789:ABCdef...`)
3. В Agent Lab откройте настройки бота
4. В разделе "Платформы" включите Telegram
5. Вставьте токен и сохраните
6. Напишите боту в Telegram - он ответит

**Важно**: Бот будет работать в режиме long polling (не требует webhook URL)

### WhatsApp

1. Зарегистрируйтесь в Meta Business Suite
2. Создайте приложение WhatsApp Business
3. Получите:
   - Phone Number ID
   - Access Token
   - Webhook Verify Token
4. В Agent Lab включите WhatsApp
5. Укажите данные:
   - **Phone Number ID** - ID телефона
   - **Access Token** - токен доступа
   - **Verify Token** - токен верификации webhook
6. Настройте webhook в Meta:
   - URL: `https://your-domain.com/api/v1/webhook/whatsapp/{bot_id}`
   - Verify Token: тот же, что указали в Agent Lab

### Web Chat

Встраиваемый виджет для вашего сайта.

1. В настройках бота включите Web Chat
2. Получите код виджета:

```html
<script>
  window.agentLabConfig = {
    botId: 'your_bot_id',
    apiUrl: 'https://your-domain.com'
  };
</script>
<script src="https://your-domain.com/static/chat-widget.js"></script>
```

3. Вставьте код перед закрывающим тегом `</body>`
4. Виджет появится в правом нижнем углу

### AmoCRM

Интеграция с CRM системой:

1. В AmoCRM создайте интеграцию
2. Получите код интеграции
3. В Agent Lab включите AmoCRM
4. Укажите:
   - **Subdomain** - ваш поддомен AmoCRM
   - **Integration ID** - ID интеграции
   - **Secret Key** - секретный ключ
5. Настройте виджет в AmoCRM для отображения чата

### API

Прямой доступ к боту через REST API:

```bash
curl -X POST https://your-domain.com/api/v1/flow/{bot_id}/invoke \
  -H "Authorization: Bearer YOUR_TOKEN" \
  -H "Content-Type: application/json" \
  -d '{
    "message": "Привет!",
    "user_id": "user123"
  }'
```

## Тестирование бота

### Через веб-виджет

1. В любом месте Agent Lab нажмите на иконку чата (правый нижний угол)
2. Выберите вашего бота
3. Напишите сообщение
4. Получите ответ

### Через Telegram/WhatsApp

Просто напишите боту в мессенджере после подключения.

### Через API

Используйте curl или Postman для отправки запросов.

## Мониторинг

### Просмотр диалогов

1. Перейдите в раздел "Чаты"
2. Увидите список всех диалогов
3. Кликните на диалог для просмотра полной переписки

### История запросов

1. Перейдите в раздел "История"
2. Фильтруйте по:
   - Боту
   - Платформе
   - Дате
   - Пользователю
3. Просматривайте детали каждого запроса:
   - Входные данные
   - Ответ агента
   - Расход токенов
   - Время выполнения

### Биллинг

1. Перейдите в раздел "Биллинг"
2. Просматривайте:
   - Расход токенов по ботам
   - Стоимость запросов
   - Вызовы внешних API
3. Устанавливайте лимиты расходов

## Редактирование бота

1. В списке ботов нажмите на карточку бота
2. Измените необходимые параметры
3. Нажмите "Сохранить"
4. Изменения применятся мгновенно

## Удаление бота

1. Откройте настройки бота
2. Нажмите "Удалить бота"
3. Подтвердите удаление

**Важно**: При удалении бота:
- История диалогов сохраняется
- Подключения к платформам отключаются
- Агенты остаются в системе (можно использовать в других ботах)

## Клонирование бота

Для создания копии существующего бота:

1. Откройте настройки бота
2. Нажмите "Клонировать"
3. Введите новое название
4. Бот будет создан со всеми настройками оригинала
5. Измените то, что нужно

## Экспорт/Импорт конфигурации

**Экспорт:**
1. Откройте настройки бота
2. Нажмите "Экспорт конфигурации"
3. Скачается JSON файл с настройками

**Импорт:**
1. Нажмите "Импорт конфигурации"
2. Выберите JSON файл
3. Проверьте настройки
4. Сохраните

Это удобно для переноса ботов между средами (dev/staging/production).

## Частые вопросы

**Бот не отвечает в Telegram**
- Проверьте правильность токена
- Убедитесь, что бот запущен (статус "Active")
- Проверьте логи в разделе "История"

**Бот дает неправильные ответы**
- Улучшите промпт агента
- Добавьте примеры в промпт
- Загрузите документы в базу знаний
- Проверьте настройки RAG

**Превышен лимит токенов**
- Перейдите в раздел "Биллинг"
- Увеличьте лимит
- Или оптимизируйте промпт (сделайте короче)

**Бот работает медленно**
- Уменьшите параметр `max_tokens` в настройках LLM
- Используйте более быструю модель (например, `gpt-3.5-turbo` вместо `gpt-4`)
- Проверьте timeout настройки
