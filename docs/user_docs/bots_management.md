# 🤖 Управление ботами :material-robot-outline:

Раздел **"Боты"** — это центр управления вашими ИИ-агентами. Здесь вы создаете, настраиваете и подключаете ботов к различным платформам.

## 🔍 Что такое бот в Agent Lab

Бот в Agent Lab — это **точка входа (Flow)**, которая объединяет несколько ключевых компонентов:

**Основные компоненты:**
- 🧠 **Агент или цепочка агентов** — логика обработки запросов
- 🌐 **Подключения к платформам** — Telegram, WhatsApp, Web, AmoCRM, API
- ⚙️ **Конфигурация и переменные** — настройки поведения
- 📚 **База знаний и LLM** — модели ИИ и данные для ответов

!!! info "Архитектура"
    Каждый бот работает как независимый поток обработки, но может использовать общие ресурсы компании.

## 📊 Просмотр списка ботов

При переходе в раздел **"Боты"** вы увидите список всех ваших ботов в виде карточек.

Каждая карточка содержит:
- **Название бота**
- **Краткое описание**
- **Подключенные платформы** (иконки Telegram, WhatsApp, Web, API, AmoCRM)
- **Entry Point Agent** (идентификатор главного агента)

Вы можете нажать на карточку бота, чтобы перейти к его детальным настройкам, или нажать на иконку чата для быстрого тестирования в веб-виджете.

## 🛠️ Редактирование бота

Чтобы изменить настройки существующего бота:

1. В списке ботов нажмите на карточку нужного бота.
2. Откроются детальные настройки бота с несколькими вкладками:
   - **Основное**: Название, описание, промпт агента, таймауты, количество повторов, опция включения размышлений (reasoning).
   - **Способности**: Управление инструментами и агентами, которые может использовать бот.
   - **База знаний**: Настройки RAG, скоупы хранения и поиска документов, а также возможность загрузки документов.
   - **Каналы**: Подключение и настройка различных каналов взаимодействия (Telegram, WhatsApp, Web, API, AmoCRM).
   - **Дополнительно**: Системная информация (Flow ID, Агент точки входа).
3. Внесите необходимые изменения.
4. Нажмите кнопку **"Сохранить"** (иконка дискеты в правом верхнем углу) для применения изменений. Изменения вступают в силу немедленно.

## ➕ Создание нового бота

Для создания нового бота смотрите раздел **[Создание ботов через интерфейс](bots_creation.md)**.

## 🗑️ Удаление бота

Чтобы удалить бота:

1. Откройте детальные настройки бота.
2. Найдите кнопку **"Удалить бота"** (обычно в правом верхнем углу или на вкладке "Дополнительно").
3. Подтвердите удаление во всплывающем окне.

!!! warning "Важно"
    При удалении бота его конфигурация и подключенные платформы будут удалены. Однако история диалогов сохранится в разделе **"Чаты"**, а агенты, если они не были уникальными для этого flow, останутся в системе и могут быть использованы в других ботах.

## 🔄 Клонирование бота

Если вы хотите создать копию существующего бота с его настройками:

1. Откройте детальные настройки бота.
2. Нажмите кнопку **"Клонировать"**.
3. Введите новое название для клонированного бота.
4. Новый бот будет создан со всеми настройками оригинала, после чего вы сможете изменить их по своему усмотрению.

## ⬆️⬇️ Экспорт и импорт конфигурации

Для переноса конфигурации бота между различными средами (разработка, тестирование, продакшн) или для резервного копирования вы можете использовать функции экспорта и импорта.

### Экспорт:
1. Откройте детальные настройки бота.
2. Нажмите кнопку **"Экспорт конфигурации"**.
3. Будет скачан JSON файл, содержащий полную конфигурацию вашего бота.

### Импорт:
1. На странице списка ботов или при создании нового бота найдите опцию **"Импорт конфигурации"**.
2. Выберите JSON файл с конфигурацией бота.
3. Система предложит вам просмотреть импортированные настройки, после чего вы сможете сохранить нового бота с этими параметрами.

## 🧪 Тестирование и отладка

### Веб-чат виджет

В правом нижнем углу интерфейса Agent Lab доступен веб-чат виджет для быстрого тестирования любого бота.
1. Нажмите на иконку чата.
2. Выберите нужного бота из списка.
3. Начните диалог, чтобы проверить его поведение.

### Просмотр истории

Все диалоги с ботами сохраняются в разделе **"Чаты"**, где вы можете просмотреть полную переписку, найти конкретные сообщения и получить базовую аналитику.

### Статистика использования

В разделе **"История"** вы можете отслеживать детальную статистику по всем запросам к ИИ, включая расход токенов, стоимость, время выполнения и информацию об ошибках. Это полезно для отладки и оптимизации бота.
