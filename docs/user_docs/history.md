# История

Раздел **"История"** в Agent Lab предоставляет детальную информацию о каждом запросе к вашим ИИ-ботам (Flows). Это мощный инструмент для мониторинга, отладки и анализа производительности.

## 📊 Обзор раздела "История"

В "Истории" вы можете отслеживать:
- Все запросы, отправленные ботам.
- Расход токенов и стоимость каждого запроса.
- Время выполнения и возникающие ошибки.
- Детальные логи каждого шага Flow, включая вызовы инструментов и промежуточные состояния.

## 🔍 Просмотр списка сессий

При открытии раздела "История" вы увидите таблицу с сессиями Flow. Каждая сессия представляет собой уникальный диалог и содержит следующую информацию:

- **Платформа**: Канал взаимодействия (например, Web, Telegram, WhatsApp, API, AmoCRM).
- **Flow**: Название Flow (бота), с которым связана сессия.
- **Пользователь**: Идентификатор пользователя, инициировавшего сессию.
- **Статус**: Текущий статус сессии (например, `Active`, `Inactive`, `Processing`, `Waiting`).
- **Сообщений**: Общее количество сообщений в сессии.
- **Создано**: Дата и время начала сессии.
- **Действия**: Кнопка для просмотра детальной истории сообщений этой сессии.

## 🛠️ Фильтрация и поиск

Используйте панель фильтров для поиска нужных сессий:

- **Платформа**: Фильтрация по каналам общения.
- **Flow**: Поиск по названию или ID Flow.
- **Статус**: Фильтрация по статусу сессии.
- **Дата От / До**: Выбор диапазона дат создания сессии.

Нажмите **"Применить"** для использования фильтров или **"Сбросить"** для очистки.

## 💬 Детальный просмотр сообщений сессии

Для просмотра полной хронологии сообщений и событий внутри сессии:

1. В таблице сессий нажмите кнопку с иконкой глаза в столбце **"Действия"**.
2. Откроется модальное окно "История диалога", где представлены все сообщения и системные события в хронологическом порядке:
   - **Сообщения пользователя**: Ваши запросы к боту.
   - **Ответы ассистента**: Сгенерированные ответы бота.
   - **Вызовы инструментов**: Информация о том, какие инструменты и с какими аргументами были вызваны ботом.
   - **Системные сообщения**: Внутренние события Flow, такие как изменение состояния или ошибки.
   - **Время и источник**: Для каждого сообщения указывается время и агент (если применимо), который его сгенерировал.

## ⚙️ Анализ и отладка

Раздел "История" — незаменимый инструмент для:
- **Отладки Flow**: Помогает понять, как бот обрабатывает запросы, почему возникают ошибки или нежелательное поведение.
- **Мониторинга производительности**: Отслеживание времени выполнения и использования ресурсов LLM.
- **Оптимизации затрат**: Анализ расхода токенов для каждого запроса.
- **Анализа взаимодействия**: Глубокое понимание пользовательских паттернов и потребностей.
