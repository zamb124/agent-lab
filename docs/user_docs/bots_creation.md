# Создание ботов через интерфейс :material-robot-happy-outline:

Этот гайд научит вас создавать собственных ИИ-ботов через веб-интерфейс Agent Lab. Вы сможете настроить поведение, подключить платформы и запустить бота без программирования.

## 🚀 Создание нового бота

### Шаг 1: Открываем раздел Боты

В левом меню нажмите **"Боты"**.

### Шаг 2: Начинаем создание

Нажмите **"+ Создать бота"** или кнопку **"Создать бота"** на странице списка ботов.

### Шаг 3: Заполняем основные поля

На вкладке **"Основное"** укажите базовую информацию о вашем боте:

- **Название бота** - понятное имя вашего бота (обязательное поле). На основе названия будет автоматически сгенерирован Flow ID и Agent ID.
- **Описание** - краткое описание назначения бота (обязательное поле).
- **Таймаут (сек)** - максимальное время ожидания ответа от бота (в секундах). Если бот не ответит в течение этого времени, сессия может быть прервана.
- **Максимум повторов** - количество попыток, которые бот предпримет в случае возникновения ошибки перед тем, как завершить работу или сообщить об ошибке.
- **Промпт агента** - ключевая инструкция для ИИ-модели, определяющая его поведение и характер. Здесь вы описываете, как должен отвечать ваш бот, какие задачи выполнять и в каком стиле общаться.
- **Включить размышления (reasoning)** - опция для отправки промежуточных размышлений LLM пользователю. Работает только с моделями, поддерживающими эту функцию (например, Google Gemini 2.5 Pro).

## 🧠 Настройка способностей (Агенты и Инструменты)

На вкладке **"Способности"** вы управляете инструментами и агентами, которые может использовать ваш бот.

**Инструменты** позволяют боту взаимодействовать с внешними системами и выполнять специфические действия (например, поиск в интернете, выполнение API-запросов, сохранение данных).

**Агенты-инструменты** (помечены 🤖) - это другие боты, которые могут быть вызваны текущим ботом для выполнения подзадач, создавая многоагентные системы.

Вы можете добавлять, удалять и настраивать каждый инструмент или агента, доступного вашему боту.

## 📚 Настройка Базы знаний (RAG)

На вкладке **"База знаний"** вы можете подключить к боту документы и файлы для улучшения качества ответов с помощью RAG (Retrieval-Augmented Generation). Бот будет автоматически искать релевантную информацию в этих документах.

**Параметры RAG:**
- **Скоуп хранения** - определяет, где будут храниться документы, загруженные через бота:
  - `Компания` (общая база для всех flow в рамках вашей компании)
  - `Flow` (личная база только для этого бота)
  - `Сессия` (временные документы, привязанные к конкретной сессии диалога)
- **Скоупы поиска** - указывает, где бот будет искать информацию при запросах:
  - `Искать в общей базе компании`
  - `Искать в базе этого flow`
  - `Искать в базе текущей сессии`
- **Документы в базе знаний** - здесь вы можете загружать PDF, TXT, DOCX и другие форматы документов, которые бот будет использовать для ответов.

## 🌐 Подключение платформ

На вкладке **"Каналы"** вы настраиваете каналы общения бота с пользователями. Вы можете подключить бота к различным мессенджерам, веб-чату или REST API.

**Доступные платформы:**
- **Telegram**
- **WhatsApp**
- **Web Chat**
- **REST API**
- **AmoCRM**

Для каждой платформы предусмотрены свои настройки подключения (например, токены, ID).

## ⚙️ Дополнительные настройки

На вкладке **"Дополнительно"** вы найдете системную информацию о боте:
- **Flow ID** - уникальный системный идентификатор вашего flow.
- **Агент точки входа** - системный идентификатор агента, который обрабатывает входящие запросы к этому боту.

## ✅ Сохранение бота

После заполнения всех необходимых полей и настройки, нажмите кнопку **"Сохранить"** (иконка дискеты в правом верхнем углу), чтобы применить изменения.

## 🧪 Тестирование бота

Вы можете протестировать своего бота несколькими способами:
- **Через веб-виджет:** В правом нижнем углу интерфейса Agent Lab находится виджет чата. Выберите своего бота и начните диалог.
- **Через подключенные платформы:** Если вы подключили Telegram или WhatsApp, напишите боту напрямую в мессенджере.
- **Через API:** Если вы настроили API-интеграцию, используйте `curl` или другой HTTP-клиент для отправки тестовых запросов.
