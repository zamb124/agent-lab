# Устранение проблем :material-bug-outline:

В этом разделе собраны решения наиболее частых проблем, с которыми вы можете столкнуться при работе с платформой Agent Lab. Если вы не нашли ответ на свой вопрос, обратитесь в [службу поддержки](getting_started.md#нужна-помощь).

## :material-alert-circle-outline: Общие проблемы

### :material-robot-off-outline: Бот не отвечает

**Возможные причины:**
- Неправильные API ключи или токены.
- Недостаточно средств на балансе компании.
- Проблемы с подключением к платформам.
- Ошибки в промпте или логике Flow.

**Решения:**
- Проверьте настройки API ключей и токенов в разделе [Ключи и Переменные](variables.md).
- Проверьте баланс компании в разделе [Биллинг](billing.md).
- Убедитесь, что платформы подключены и активны ([Платформы](platforms.md)).
- Посмотрите детальные логи бота в разделе [История](history.md).

### :material-speedometer-slow: Медленные ответы

**Оптимизации:**
- Выберите более быструю LLM-модель (например, Flash версии) для вашего бота.
- Сократите длину промпта агента, убрав избыточную информацию.
- Проверьте лимиты и квоты, установленные для ваших API ключей.
- Рассмотрите возможность кэширования для часто повторяющихся запросов.

### :material-currency-off: Ошибки биллинга

**`BillingError` (недостаточно средств):**
- Проверьте текущий баланс компании в разделе [Биллинг](billing.md).
- Пополните баланс при необходимости.

**`TariffError` (ресурс недоступен на тарифе):**
- Проверьте доступность используемого ресурса (например, конкретной LLM-модели или инструмента) для вашего текущего тарифного плана.
- Обратитесь в поддержку для обсуждения повышения тарифа.

## :material-connection-alert: Проблемы по платформам

### :material-telegram: Telegram
- **Неправильный токен**: Убедитесь, что токен BotFather введен корректно и не содержит лишних символов. Получите новый токен, если есть сомнения.
- **Бот заблокирован**: Проверьте статус бота в BotFather. Возможно, он был заблокирован из-за нарушений правил.
- **Права доступа**: Убедитесь, что бот имеет необходимые права в чате (если это группа).

### :material-whatsapp: WhatsApp
- **Настройки Webhook**: Проверьте правильность URL webhook и токена верификации в Meta Business Suite. Убедитесь, что `hub.mode` и `hub.verify_token` соответствуют.
- **Невалидные токены Meta**: Access Token может быть просрочен или недействителен. Обновите его.
- **Статус номера телефона**: Проверьте статус вашего номера WhatsApp Business в Meta. Он должен быть верифицирован.
- **Шаблоны сообщений**: Для исходящих сообщений вне 24-часового окна поддержки требуются предварительно одобренные шаблоны.

### :material-web: Web Chat
- **HTTPS сертификат**: Убедитесь, что ваш сайт использует HTTPS. Web Chat виджет не будет работать на HTTP-сайтах.
- **Код виджета**: Проверьте, что код виджета корректно вставлен перед закрывающим тегом `</body>` и содержит правильный `botId` и `apiUrl`.
- **CORS настройки**: Если вы используете собственный домен для Agent Lab, убедитесь, что настройки CORS на вашем сервере позволяют загрузку скрипта виджета.
