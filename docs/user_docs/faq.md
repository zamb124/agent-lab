# Часто задаваемые вопросы (FAQ) :material-help-box-outline:

Здесь собраны ответы на наиболее частые вопросы по использованию платформы Agent Lab.

## :material-comment-question-outline: Общие вопросы

### :material-robot-happy-outline: Что такое Agent Lab?
Agent Lab - это современная платформа для создания и управления ИИ-агентами без навыков программирования. Она позволяет автоматизировать бизнес-процессы и создавать интеллектуальных помощников.

### :material-rocket-launch-outline: Как начать работу?
1. Зарегистрируйтесь через Yandex или Google.
2. Создайте компанию.
3. Установите API ключи (если требуются).
4. Создайте первого бота из [Магазина ботов](store.md).

### :material-currency-usd: Сколько стоит использование?
Agent Lab работает по модели **pay-as-you-go** (плати по факту использования). Есть бесплатный тариф с лимитами. Подробности смотрите в разделе [Биллинг](billing.md).

## :material-code-tags: Технические вопросы

### :material-brain: Какие ИИ модели поддерживаются?
Поддерживаются популярные LLM-модели от разных провайдеров, такие как OpenAI GPT-4, YandexGPT, Google Gemini и другие. Вы можете выбрать подходящую модель при создании или настройке бота.

### :material-connection: Какие платформы доступны?
Доступны следующие платформы для подключения ботов: Telegram, WhatsApp Business, Web-виджеты, REST API и AmoCRM. Подробности в разделе [Платформы](platforms.md).

### :material-cog-outline: Как настроить интеграцию?
Подробные инструкции по настройке каждой платформы и интеграций находятся в разделе [Платформы](platforms.md).

## :material-bug-outline: Проблемы и решения

### :material-robot-off-outline: Бот не отвечает
Проверьте:
- Подключены ли платформы и активны ли они.
- Настроены ли все необходимые API ключи и токены.
- Правильно ли составлен промпт агента.
- Логи бота в разделе [История](history.md) на предмет ошибок.

### :material-cash-remove: Высокая стоимость
Для оптимизации расходов:
- Используйте более экономичные LLM-модели (например, Flash версии) для простых задач.
- Сокращайте длину входных и выходных промптов.
- Рассмотрите возможность использования кэширования для повторяющихся запросов.
- Настройте лимиты расходов в разделе [Биллинг](billing.md).
