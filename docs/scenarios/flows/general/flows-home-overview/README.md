# Flows: главная страница и обзор сервиса

Главная `/flows/` — рабочий обзор сервиса. Здесь разработчик видит доступные flow, быстрые действия, состояние runtime-контуров и переходы в чат или редактор.

## Шаг 1. Открываем Flows

Откройте `/flows/`. Слева находится каталог flow с поиском и кнопкой создания. В центральной области видны hero-блок, метрики, быстрые действия и карточки текущих flow.

![Главная страница Flows в светлой теме: слева каталог flow, в центре обзор, метрики, быстрый старт и карточки агентов.](screenshots/001.png)

## Что проверяем на главной

- **Каталог слева** — поиск по flow, быстрый переход в нужного агента, кнопка `+` для создания.
- **Создать flow** — открывает мастер создания flow или установку bundle из Store.
- **Сессии** — открывает историю запусков и позволяет продолжить диалог.
- **MCP** — открывает настройки MCP-серверов и синхронизированных tools.
- **Метрики** — показывают число flow, веток, нод, активных сессий и bundle updates.
- **Ваши flows** — карточки агентов с кнопками открытия чата и редактора.
- **Контуры подключения** — переменные, integrations, resources и MCP как отдельные рабочие поверхности.
- **Живая работа** — последние сессии, readiness tools/bundles и операторский контур.

## Когда использовать этот экран

Используйте главную как стартовую точку перед любой работой с FLOWS:

1. Найти существующий flow и открыть чат.
2. Открыть редактор для изменения графа.
3. Создать новый flow.
4. Проверить, что MCP, integrations, variables и resources настроены.
5. Продолжить недавнюю сессию или открыть операторский контур.
