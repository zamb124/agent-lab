# Platform basics: entry, Dashboard, and user menu

A basic path for a new user: open the website, go to Dashboard, understand the service list, and inspect the user menu.

## Step 1. Open the main website. If the user is signed in, the header has a Dashboard button.

![Open the main website. If the user is signed in, the header has a Dashboard button.](screenshots/001.png)

## Step 2. Dashboard shows the company's main services. System-only services are hidden from a regular company.

![Dashboard shows the company's main services. System-only services are hidden from a regular company.](screenshots/002.png)

## Step 3. Open the user menu. It contains apps, profile, company, calendar, documentation, language, theme, and sign out.

![Open the user menu. It contains apps, profile, company, calendar, documentation, language, theme, and sign out.](screenshots/003.png)

## Step 4. The company row opens the company list. The checkmark shows the active company, and Create company starts a new company.

![The company row opens the company list. The checkmark shows the active company, and Create company starts a new company.](screenshots/004.png)

## Step 5. Apps opens the service launcher. It is a quick way to go to Flows, NetWorkle, RAG, Sync, or Documents.

![Apps opens the service launcher. It is a quick way to go to Flows, NetWorkle, RAG, Sync, or Documents.](screenshots/005.png)

## Step 6. Profile opens the user card with name, email, roles, and account actions.

![Profile opens the user card with name, email, roles, and account actions.](screenshots/006.png)

## Step 7. The lower menu items: Documentation opens help, Language changes the locale, Theme switches appearance, and Logout signs out.

![The lower menu items: Documentation opens help, Language changes the locale, Theme switches appearance, and Logout signs out.](screenshots/007.png)

## Step 8. Calendar opens the fullscreen calendar. The modal stays inside the viewport and is ready to use.

![Calendar opens the fullscreen calendar. The modal stays inside the viewport and is ready to use.](screenshots/008.png)
