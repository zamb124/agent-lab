# Директория переводов

Эта директория содержит все файлы системы интернационализации (i18n) для Agents Lab.

## 📁 Структура

```
app/i18n/
├── translations/               # Основные файлы переводов (модульная структура)
│   ├── ru/                    # Русский язык
│   │   ├── _meta.json         # Метаданные языка (версия, статистика)
│   │   ├── common.json        # Общие переводы (save, cancel, delete...)
│   │   ├── dashboard.json     # Дашборд
│   │   ├── landing.json       # Лендинг
│   │   ├── bots.json          # Боты
│   │   ├── chats.json         # Чаты
│   │   ├── builder.json       # Agent Builder
│   │   ├── billing.json       # Биллинг и тарифы
│   │   ├── admin.json         # Админ панель
│   │   ├── auth.json          # Авторизация
│   │   ├── validation.json    # Валидация
│   │   ├── errors.json        # Ошибки
│   │   ├── languages.json     # Названия языков
│   │   ├── misc.json          # Прочие ключи
│   │   └── models/            # Переводы для моделей
│   │       ├── agent.json
│   │       ├── flow.json
│   │       ├── user.json
│   │       ├── context.json
│   │       └── ...
│   │
│   ├── en/                    # Английский язык (такая же структура)
│   │   ├── _meta.json
│   │   ├── common.json
│   │   ├── dashboard.json
│   │   └── ...
│   │
│   └── translations_backup/   # Бэкап старых монолитных файлов
│       ├── ru.json
│       └── en.json
│
├── keys/                      # Служебные файлы (автогенерированные)
│   ├── models.json            # Ключи из Pydantic моделей
│   ├── templates.json         # Ключи из HTML шаблонов
│   └── frontend.json          # Ключи из JavaScript файлов
│
├── generated/                 # JS модули для фронтенда (автогенерированные)
│   ├── ru.js                  # JavaScript модуль с переводами на русском
│   └── en.js                  # JavaScript модуль с переводами на английском
│
└── README.md                  # Этот файл
```

## 🔄 Автоматическая генерация

Все файлы в этой директории **автоматически генерируются** при запуске приложения:

1. **Сканирование кода** - система находит все ключи переводов в:
   - Pydantic моделях с `Field(title=...)`
   - HTML шаблонах с `{{ t('ключ') }}`
   - JavaScript файлах с `app.i18n.t('ключ')`

2. **Обновление JSON файлов** - новые ключи добавляются в `/translations/`

3. **Генерация JS модулей** - создаются оптимизированные модули в `/generated/`

## 📝 Редактирование переводов

### Модульная структура

Переводы организованы в небольшие модули по функциональным областям:

- **Маленькие файлы** (50-200 строк) вместо гигантских JSON
- **Логичная структура** - легко найти нужный перевод
- **Меньше конфликтов** - каждый работает со своим модулем
- **Быстрая загрузка** - можно загружать только нужные модули

### Ручное редактирование

Редактируйте нужный модуль напрямую:

```json
// Пример: app/i18n/translations/en/common.json
{
  "save": "Save",                      // ✅ Переведено
  "cancel": "[TODO: common.cancel]",   // ❌ Требует перевода
  "delete": "Delete"
}
```

```json
// Пример: app/i18n/translations/en/models/agent.json
{
  "fields": {
    "name": {
      "title": "Agent Name",
      "description": "Unique name for the agent"
    }
  }
}
```

**Правила:**
- ✅ **Можно изменять** значения переводов в любом модуле
- ✅ **Можно создавать** новые модули для новых разделов
- ❌ **Нельзя удалять** файл `_meta.json`
- 💡 **Совет:** работайте с небольшими модулями вместо больших файлов

### Маркировка переводов

- `[TODO: ключ]` - требует перевода
- Обычный текст - переведено
- `{параметр}` - подстановочные параметры

### Соответствие ключей и модулей

- `common.save` → `translations/ru/common.json`
- `dashboard.title` → `translations/ru/dashboard.json`
- `models.agent.fields.name` → `translations/ru/models/agent.json`
- `billing.current_plan.title` → `translations/ru/billing.json`

## 🚀 Миграция на модульную структуру

Если у вас есть старые монолитные файлы переводов, выполните миграцию:

```bash
# Запустить скрипт миграции
uv run python migrate_translations.py
```

Скрипт автоматически:
1. Создаст бэкап старых файлов в `translations_backup/`
2. Разобьет монолитные `ru.json` и `en.json` на модули
3. Создаст новую структуру директорий `ru/` и `en/`
4. Сохранит метаданные в `_meta.json`

**После миграции:**
- Старые файлы сохраняются в бэкапе
- Система автоматически использует новую модульную структуру
- Обратная совместимость сохраняется (можно работать с обеими структурами)

## 🚫 Что НЕ нужно редактировать

- **`/keys/`** - служебные файлы, перезаписываются при каждом запуске
- **`/generated/`** - JS модули, автогенерируются из JSON файлов
- **`_meta.json`** - метаданные файлов, вычисляются автоматически
- **`translations_backup/`** - бэкап старых файлов

## 🔧 Управление переводами

### Принудительная генерация

```bash
# Через API (нужны права админа)
curl -X POST http://localhost:8001/frontend/api/i18n/refresh

# При запуске приложения (автоматически)
uv run python run.py
```

### Статистика переводов

```bash
# Через API
curl http://localhost:8001/frontend/api/i18n/stats

# В консоли браузера
fetch('/frontend/api/i18n/stats').then(r => r.json()).then(console.log)
```

### Проверка полноты переводов

```bash
# Поиск непереведенных ключей
grep -r "\[TODO:" app/i18n/translations/

# Подсчет процента завершенности
jq '.meta.completeness' app/i18n/translations/en.json
```
