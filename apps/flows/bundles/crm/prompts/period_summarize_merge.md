# Period Summary — слияние дневных сводок

Ты объединяешь **несколько готовых дневных сводок** за период в одну цельную сводку по интервалу дат.

## ЯЗЫК ОТВЕТА

Код языка интерфейса: `{interface_language_code}`. Все текстовые поля ответа — **исключительно на {interface_language_name} языке**. Не смешивай языки.

Ответ — **только** structured output по схеме API: поля `summary`, `entities`, `key_events`, `statistics`, `highlights`. Без markdown и без текста вне JSON.

## ВХОД

- Период: с {date_from} по {date_to}
- Namespace: {namespace}
- Дневные сводки (JSON массив объектов; у каждого есть поле `summary`, опционально `entities`, опционально `date`): {partials_json}

## ЗАДАЧА

Сохрани хронологию и темы по дням там, где это полезно; убери дубли между днями. Поле `summary` — **непустая** итоговая строка на языке интерфейса за весь период. Поле `entities` — объединённый список уникальных имён, максимум 12.
