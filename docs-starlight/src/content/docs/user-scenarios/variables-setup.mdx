# Создание и настройка переменных

Переменные позволяют безопасно хранить API ключи, токены и другие конфиденциальные данные.

<Steps>

1. **Откройте раздел Переменные**
   
   Откройте раздел **Переменные** в боковом меню. Здесь отображается список всех переменных компании.
   
   ![Открытие раздела Переменные](/src/assets/variables-setup/01.png)

2. **Создайте переменную**
   
   Нажмите на карточку **Создать переменную** для добавления новой переменной.
   
   ![Создание переменной](/src/assets/variables-setup/02.png)

3. **Откройте форму**
   
   Откроется модальное окно для ввода данных переменной.
   
   ![Форма создания](/src/assets/variables-setup/03.png)

4. **Введите ключ**
   
   Введите **ключ переменной** — это уникальный идентификатор. Используйте snake_case без пробелов.
   
   ![Ввод ключа](/src/assets/variables-setup/04.png)

   <Aside type="tip">
   Примеры ключей: `telegram_bot_token`, `openai_api_key`, `database_url`
   </Aside>

5. **Добавьте описание**
   
   Добавьте **описание** переменной, чтобы потом было понятно её назначение.
   
   ![Добавление описания](/src/assets/variables-setup/05.png)

6. **Введите значение**
   
   Введите **значение** переменной. Для API ключей и токенов это обычно длинная строка.
   
   ![Ввод значения](/src/assets/variables-setup/06.png)

7. **Установите флаг секрета**
   
   Отметьте чекбокс **Секрет**, если значение должно быть скрыто (для паролей и токенов).
   
   ![Установка флага секрета](/src/assets/variables-setup/07.png)

   <Aside type="caution">
   Секретные переменные нельзя просмотреть после сохранения — только перезаписать.
   </Aside>

8. **Сохраните переменную**
   
   Нажмите кнопку **сохранения** (иконка дискеты) для создания переменной.
   
   ![Сохранение переменной](/src/assets/variables-setup/08.png)

9. **Готово!**
   
   Переменная создана! Теперь её можно использовать в конфигурации ботов.
   
   ![Переменная создана](/src/assets/variables-setup/09.png)

</Steps>

## Использование переменных

Для использования переменной в настройках бота или MCP сервера используйте синтаксис:

```
@var:имя_переменной
```

Например:
- `@var:telegram_bot_token` — для токена Telegram бота
- `@var:openai_api_key` — для API ключа OpenAI
